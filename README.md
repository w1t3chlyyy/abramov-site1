# Сайт — Егор Абрамов / Бизнес под ключ

Node.js-сервис: отдаёт статический сайт (`public/index.html`) и принимает заявки с формы
через `POST /api/lead`, которые пересылаются в Telegram-бота. Токен бота хранится
только в переменных окружения — в коде и в браузере его больше нет.

## ⚠️ Обязательно перед деплоем

В исходном файле токен бота был прописан прямо в HTML/JS — это значит, что он мог
"засветиться". Прежде чем публиковать сайт:

1. Откройте чат с **@BotFather** → `/mybots` → выберите бота → **Bot Settings → API Token → Revoke current token**
2. Скопируйте новый токен — он понадобится на шаге деплоя ниже
3. Chat ID можно оставить прежний (если это ваш личный чат/группа), либо получить заново,
   написав боту `/start` и открыв `https://api.telegram.org/bot<НОВЫЙ_ТОКЕН>/getUpdates`

## Деплой на Render.com

1. Залейте эту папку в репозиторий на GitHub/GitLab:
   ```bash
   git init
   git add .
   git commit -m "initial commit"
   git remote add origin https://github.com/ваш-логин/abramov-site.git
   git branch -M main
   git push -u origin main
   ```
2. На render.com: **New → Blueprint** → выберите репозиторий
   (Render прочитает `render.yaml` и создаст веб-сервис `sokolov-business`)
3. При создании (или потом в **Environment**) укажите переменные:
   - `TELEGRAM_BOT_TOKEN` — новый токен бота
   - `TELEGRAM_CHAT_ID` — id чата/группы, куда должны приходить заявки
4. Нажмите **Apply / Create** — через пару минут сайт будет доступен на
   `https://sokolov-business.onrender.com`

## Локальный запуск (проверить перед деплоем)

```bash
npm install
cp .env.example .env   # и вписать туда реальные значения
node server.js
```
Открыть `http://localhost:3000` — форма внизу страницы отправляет заявку в бота.

## Что ещё стоит поправить перед публикацией
- Контакты в разделе "Контакты" (email/телефон) — на реальные, если там ещё примеры
- Ссылка на Hustlify (`#hustlify` → `https://hustlify.ru`) — уже настоящая, проверьте, что сайт работает
- На бесплатном плане Render веб-сервис "засыпает" при простое — первый запрос после паузы
  может грузиться 30–50 секунд. Если это критично — нужен платный план.
